Projektor

↑ {% trans "Nach oben" %}

Automatische Projektoransicht

Der Projektor aktualisiert sich jede Sekunde automatisch. Änderung an der Tagesordnung oder anderen Elementen von OpenSlides werden so unmittelbar am Projektor sichtbar. Technisch läuft der Projektor im Vollbildmodus des Browsers; z.B. auf einem erweiterten zweiten Monitor oder auf einem anderen Conmputer.

Eigene Folien erstellen

Über das Dashboard können Sie benutzerdefinierte Folien anlegen, die sich gut für Themen abseits der dynamischen Folien zu Tagesordnung, Anträgen und Wahlen eignen.

Dashboard

Alle Folien (zu Tagesordnung, Anträgen und Wahlen) werden auf dem Dashboard übersichtlich in zusammenklappbaren Widgets aufgelistet. Eine verkleinerte Projektor-Live-Ansicht zeigt immer die aktuell ausgewählte Folie an.

Uhrzeit und Countdown

Alle Teilnehmer haben die aktuelle Uhrzeit auf der Projektoransicht jederzeit im Blick. Für eine Redezeitbeschränkung bietet OpenSlides einen Countdown. Die Startzeit lässt sich frei definieren.

Überblendungen

Overlay-Nachrichten lassen sich über normale Folien einblenden. Dies kann sinnvoll sein, wenn z.B. organisatorische Informationen angezeigt, die aktuelle Folie aber nicht zu verlassen werden soll.




Tagesordnung

Tagesordnung verwalten

Jeder Tagesordungseintrag besteht aus einem Titel sowie - optional - aus einem zusätzlichen Text und einem internen Kommentar (z.B. für Regieanweisungen der Versammlungsleitung). Im Text können HTML-Tags verwendet werden, um z.B. Bilder oder Videos einzubetten.

Am Projektor anzeigen

Jeder Tagesordnungseintrag lässt sich per Mausklick auf dem Projektor anzeigen. Egal in welcher Reihenfolge!

Tagesordnung sortieren

Einträge lassen sich in der Tagesordnung frei per Drag&Drop sortieren - mit beliebig vielen Unterpunkten.

Erledigt markieren

Abgeschlossene Tagesordnungspunkte können als erledigt markiert und anschließend zur besseren Übersicht ausgeblendet werden.

PDF

Die vollständige Tagesordnung lässt sich als PDF erzeugen.




Anträge

↑ {% trans "Nach oben" %}

Anträge verwalten

Jeder Antrag besteht aus Titel, Text und Antragsteller sowie - optional - aus einer zusätzlicher Begründung und Unterstützerliste.

Zugriffsrechte für Anträge regeln

Über ein Rechtemanagement können Sie genau entscheiden, welche Benutzergruppen Anträge sehen, erstellen, verwalten und unterstützen darf.

Unterstützen von Anträgen

Sie können entscheiden, wie viele Unterstützer nötig sind, um einen Antrag zuzulassen. Teilnehmer können direkt in OpenSlides Ihre Zustimmung für einen Antrag äußern. Abschalten lässt sich dieses Feature natürlich auch.

Anträge in die Tagesordnung

Jeder Antrag lässt sich als Referenz zusätzlich in die Tagesordnung einsortieren.

Antragsstatus ändern

Jeder Antrag durchläuft verschiedene Phasen in demokratischen Versammlungen. OpenSlides bildet diese ab; z.B. angefangen bei noch nicht zugelassen und auf Unterstützersuche bis hin zu angenommen, abgelehnt, vertagt oder zurückgezogen.

Anträge abstimmen

Nach Abstimmung eines Antrags im Plenum wird das ausgezählte Ergebnis in OpenSlides eingegeben (mit OpenSlides selbst wird nicht abgestimmt!). Dabei sind auch spezielle Werte für die Mehrheit der Stimmen, oder nicht erfasste Stimmen möglich. Anträge werden immer mit Ja-Nein-Enthaltung abgestimmt. Optional sind die abgegebenen Stimmen. Zusätzlich unterstützt OpenSlides ungültige Stimmen und PDF-Stimmzettel zum Ausdrucken für geheime Abstimmungen.

Versionshistorie

Nur Antragsteller und Versammlungsleiter dürfen Anträge ändern. Dabei werden Änderungen als neue Version abgespeichert. Mit der Option triviale Änderung wird keine neue Version angelegt, dies kann z.B. bei Korrektur von Tippfehlern sinnvoll sein.

Aktionen protokollieren

Jede Aktion, die Sie an einem Antrag durchführen, wird von OpenSlides protokolliert. Das bietet Ihnen als Versammlungsleiter immer die nötige Rückverfolgung, welcher Teilnehmer wann einen Antrag erstellt, bearbeiten oder dessen Status geändert hat.

Antragsbücher als PDF

Alle vorhandenen Anträge lassen sich übersichtlich in ein PDF erstellen; inkl. Informationen zu Antragsteller, Abstimmgungsergebnis und Begründung eine Antrags.




Wahlen

↑ {% trans "Nach oben" %}

Wahlen verwalten

Jede Wahl besteht aus Name und Anzahl der zur Wahl stehenden Posten sowie - optional - aus einer Beschreibung und einem Kommentar für den Stimmzettel.

Kandiatenliste aufstellen

Kandidaten können für eine Wahl direkt aus der Teilnehmerliste vorgeschlagen werden. Angemeldete Teilnehmer können per Mausklick sogar selbst kandidieren - sofern Sie die Berechtigung dazu haben.

Stimmzettel als PDF

Geheime Wahlen benötgen Stimmzettel. Und diese können Sie schnell und unkompliziert direkt mit OpenSlides generieren - mit passenden Ankreuzfeldern vor jedem Kandidaten.

Wahlergebnisse eingeben und darstellen

Sind die Wahlergebnisse ausgezählt, werden sie in OpenSlides vom Versammlungsleiter eingegeben. Per Knopfdruck (z.B. zeitgleich mit der Verkündigung des Resultats) werden die Zahlen auch für alle anderen Teilnehmer am Projektor und im Webinterface sichtbar.

Unterstützte Wahlmethoden

OpenSlides unterstützt die Auswahl von 2 Wahlmethoden: "Eine Stimme pro Kandidat/in" oder "Ja, Nein, Enthaltungs pro Kandidat/in". Eine automatische Zuordnung beider Methoden, abhängig von der Anzahl der Kandidaten und zu vergebenen Posten, ist die Voreinstellung in OpenSlides. Ungültige und abgegebene Stimmen können zusätzlich immer erfasst werden.

Mehrere Wahlgänge

Mit OpenSlides können Sie beliebig viele Wahlgänge durchführen - und diese übersichtlich nebeneinander darstellen. Auch Stichwahlen sind leicht damit möglich.




Teilnehmer

↑ {% trans "Nach oben" %}

Teilnehmer/innen verwalten

Neben Name, E-Mail und Geschlecht können Sie für jeden Teilnehmer Angaben zur zugehörigen Gruppe (d.h. eine Teilgruppe Ihrer Veranstaltung, z.B. der Ortsverband Berlin) und zum Amt/Gremium machen. Optional ist ein Kommentar für interne Notizen der Versammlungsleitung.

Benutzerrollen festlegen

Was jeder Teilnehmer sehen darf und was nicht definieren Sie mit einzelnen Rechten. Mehrere Rechteeinstellungen können Sie in einer Benutzerrolle zusammenfassen (voreingestellte Rollen sind: Gast, Beobachter, Delegierte, Versammlungsleitung, Teilnehmerverwaltung). Jeder Teilnehmer kann eine oder mehrere Benutzerrollen zugewiesen bekommen. Auch ein Gast/Anonymous-Nutzer ist so aktivierbar.

Teilnehmer/innen aus CSV-Dateien importieren

Sie können Ihre vorhandenen Teilnehmerlisten im CSV-Format abspeichern und in OpenSlides importieren. Eine Beispieldatei mit den erforderlichen Feldern gibt im OpenSlides-Wiki.
